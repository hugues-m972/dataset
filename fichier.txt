Resume soutenance

1 les variables du dataset 
1. Variables temporelles
 Elles décrivent le moment où se passe l’action

timestamp : date complète (ex: 2024-01-01 14:00)

hour : heure (0 → 23)

day_of_week : jour (0 = lundi, 6 = dimanche)

month : mois

year : année

2 Variables cycliques (très important pour LSTM)
hour_sin

hour_cos

👉 Elles servent à représenter le temps de manière continue

💡 Exemple :

23h et 0h sont proches → sin/cos permet ça

3 Consommation énergétique

Ce sont les charges du système

motor_power_W : consommation du moteur (propulsion 🚤)

camera_power_W : consommation caméra 📷

instruments_power_W : capteurs / instruments 🔬

total_power_W : somme totale consommée

Batterie

C’est le cœur énergétique du système

battery_soc : niveau de batterie (%)

battery_energy_Wh : énergie restante

battery_voltage_V : tension

battery_current_A : courant

 Énergie solaire
👉 Source d’énergie externe

solar_irradiance : intensité du soleil

solar_power_W : puissance produite

solar_energy_Wh : énergie produite

Contexte mission

 Donne le contexte d’utilisation du bateau

mission_phase :

idle (repos)

navigation

data_collection

return

criticality :

low

medium

high

👉 ça sert pour RL (prise de décision)
Variables d’action 

 Ce que le système peut contrôler

motor_active (0/1) : moteur ON/OFF

camera_active (0/1)

instruments_active (0/1)

💡 RL va décider ça


Priorités
👉 Importance des modules

motor_priority (1–3)

camera_priority (1–3)

instruments_priority (1–3)

💡 Exemple :

moteur = priorité haute (navigation critique)

Variables futures (LSTM)
👉 Pour la prédiction

future_power : consommation future

future_battery_soc : batterie future

Variables techniques
sum_modules : somme des puissances

battery_diff : variation batterie

👉 utiles pour vérification
